# OpenFang Agent ID 缓存问题 - 修复完成

## 问题

Telegram bot 发送媒体组时出现 9 次 "Agent not found" 错误，每次 OpenFang daemon 重启后都需要手动修复。

## 根本原因

OpenFang 的 reconcile 机制会定期重建 agent（生成新 UUID），但 Telegram channel bridge 缓存了旧的 agent ID，没有监听 agent 生命周期事件来更新缓存。

## 解决方案

在 `crates/openfang-api/src/channel_bridge.rs` 中添加后台任务，监听 `LifecycleEvent::Spawned` 事件，当 agent 重建时自动更新 router 缓存。

## 修改内容

1. **channel_bridge.rs**（约 40 行新增代码）
   - 添加后台监听任务
   - 使用 `set_channel_default_with_name()` 保存 agent 名称
   - Agent 重建时自动更新 router 缓存

2. **测试文件**
   - `tests/agent_respawn_test.rs` - 2 个测试用例全部通过
